## About

LeetCode is a platform for preparing technical coding interviews. This repository serves as my personal collection of solutions to LeetCode problems. Each problem's solution is organized into a separate folder, with a descriptive README file and the corresponding code.

## Usage

Feel free to use this repository as a reference or for inspiration when you're working on LeetCode problems. You can browse the folders to find solutions to specific problems.

To run the code, follow the instructions provided in each problem's README file. In most cases, the code can be executed using common programming languages such as Python, Java, or C++.

## Folder Structure

The repository is organized as follows:


Each problem folder contains:

- `README.md`: A detailed explanation of the problem, including the problem statement, constraints, and my thought process for solving it.
- `solution.<extension>`: The code solution for the problem in a specific programming language (e.g., Python, Java, C++, etc.).

## Contributing

Contributions are welcome! If you have a better solution to an existing problem or want to add a solution for a new problem, please follow these steps:

1. Fork the repository.
2. Create a new branch for your contribution: `git checkout -b feature/new-solution`.
3. Make your changes and document them in a clear and concise manner in the problem's README file.
4. Commit your changes: `git commit -m "Add new solution for problem X"`.
5. Push your changes to your forked repository: `git push origin feature/new-solution`.
6. Create a pull request to merge your changes into this repository.
